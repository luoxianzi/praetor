# 绿灯，不等于赦免

*praetor 为什么把力气花在一个验收员上，而不是换个更强的模型。*

你今天大概已经干过这事：把一个活交给编码 agent，走开，回来，它说**做完了**。测试全绿，diff 看着也合理。你就合并了。

那次合并，是一次信仰行为。这篇文章讲的就是——我们为什么不再这么信，转而造了一道"怎么劝都不松口"的闸门。

---

## 一切的起点：那个数字

我们让 praetor 的执行端（干苦力的 Codex CLI）跑真实任务，每个结果都让一个独立验收员复查。在早期实测里，**约 1/3 的无人值守执行结果过不了独立审查。**

不是有 1/3 **崩了**。它们回来时都**看着像做完了**——diff 像模像样、测试全绿、总结信心满满——照样没过审。这里的每一份，都是你会闭眼合进去、事后才发现（甚至永远发现不了）的改动。

教训不是"换个更强的模型"。更强的模型能把这个比例压低，但没法让"信仰"变安全。真正的教训是：**回来的东西需要一道闸门，而且这道闸门必须独立。**

---

## 陷阱局（闸门真见血的地方）

要说清"独立"到底买到了什么，最干净的例子是：执行端干得很好，却照样被拒——而且拒对了。

在一次军团运行里（多个工人并行，每人有一份"只准碰这些文件"的冻结清单），我们故意把其中一个工人的派工单写成：让它**顺手往 `NOTES.md` 追加一行**——而这个文件在它声明的清单（`src/delta/**`）**之外**。

工人照办了。它实现了自己的功能。它的验收测试**全绿**。代码**功能完全正确**。

而验收员——一个没看过计划、只拿到冻结标准和 diff 的全新 agent——原样返回了这句：

```
VERDICT: FAIL — NOTES.md modified outside manifest scope src/delta/**
```

分支被删除。什么都没进主干。一个测试全过、代码正确的通道，**仅仅因为多碰了一个不该碰的文件就被毙了。**

这正是让委派变安全的那条性质。绿灯说明的是代码**做了什么**；它对代码**被允许做什么**只字未提。尤其在并行场景里，"某个工人悄悄改了共享代码"恰恰是那种"自己测试全过、却把下游全搞坏"的故障。把文件清单变成冻结标准的一部分，就把这种"合并时才爆的雷"变成了一次普通、无聊的 FAIL。

---

## 验收员为什么必须是个陌生人

一个显而易见的反问：*那为什么不让做计划的自己检查？*

因为一个参与过这活的模型，是在批自己的作业。整个行业是撞了南墙才认清这点——当前的共识是：让 AI 审查它自己刚产出的东西，在架构上就站不住脚，因为模型没法客观评价自己刚写的东西。开出的药方是：**一个上下文独立、并且有权把活判死的独立验收方。**

praetor 还多走了一步——大多数方案漏掉的那步：**验收员不仅要独立于执行者，还要独立于计划者。** 如果 Claude 既写了计划、又给结果打分，那还是在批自己的作业——只是高了一层。所以这个验收员：

1. **从没看过计划对话。** 它只拿到冻结的验收标准和 diff。它不知道某个选择当初为什么那么定，所以它不会被"理由"打动——只会被"证据"打动。
2. **读的是执行端跑之前就冻结进 git 的标准。** 是带退出码的确定性命令，不是感觉。"怎么算干完"在有人看到结果之后，不能再挪动。
3. **FAIL 是有约束力的。** 不是建议、不是供计划者跟漂亮 diff 权衡的一票。FAIL 谁也推翻不了——你不行，Claude 也不行。想要"可以商量的验收员"，你要的是另一个工具，[docs/DESIGN.md](DESIGN.md) 解释了我们为什么不做。

这三条抽掉任何一条，闸门就漏。看过计划的验收员会被说服；事后才写的标准会被掰弯来迁就结果；可推翻的 FAIL 不过是一次你累了就会跳过的代码审查。

---

## 诚实的那部分：我们把自己的失败留在了 README 里

一张只写赢的成绩表，是营销，不是测量。我们的表故意把疤留着。

前四次派活里，**有两次卡死**——其中一次 29 分钟零写入地挂着——被 praetor 的硬性超时法条击杀。两次重试随后都成功了。我们没删那两行。它们就在 [README 实测表](../README.zh-CN.md#实测不吹牛)里，因为超时法条**本身就是产品**：不存在静默失败这条路，所以东西挂了你会大声知道，而不是无限等下去。

当那几次运行里我们撞上真实 bug（stderr 吞掉了重试要用的 session id；未跟踪的工具噪音溜过了基于 diff 的清单检查），**我们修的是机器，没动判决。** 验收员的职责是"对这活判得对"，不是"对我们方便"。（完整战报，连数字带失败，在 [docs/LEGION.zh-CN.md](LEGION.zh-CN.md)。）

这个区分就是整套伦理：闸门周围的机器，你可以修；为了让机器好看而放松闸门，永远不行。

---

## 这对你意味着什么

如果你本来就已经在拆活——在 Claude Code 里做计划，在 Codex 里执行，就像现在大多数人手动在做的那样——那 praetor 只是补上缺失的第三个座位：在合并之前，检查回来的东西是不是真的对得上你冻结的那份计划。

你留住了委派的速度，丢掉的是那份信仰。

- **一句话卖点：** Claude 当大脑，Codex 当双手，一个你劝不动的验收员决定什么能合并。
- **一句话记住：** 绿灯不等于赦免。没有验收的委派，只是许愿。
