# 军团模式 —— 设计理念与实战战报

*执政官指挥的是军团，复数。* 本文两件事：军团模式为什么这样设计，以及它真刀真枪跑起来发生了什么。下面每个数字都来自 2026-07-08 的真实运行（Codex CLI 0.142.5，gpt-5.5 超高档，一台笔记本），没有一处是模拟的。

## 一口气讲完设计

N 个独立 Codex 工人，每人一个**独立 git 工作树**、一份**冻结的验收标准+可碰文件清单**、一位**全新上下文的验收员**，跑完**按序合并**，最后过一道**强制集成验收**。v0.1 的三条铁律逐路生效，外加整体第四条：*集成验收不过，整个军团就不算完成。*

### 每个零件杀的是哪种事故

| 零件 | 它杀死的事故 |
|---|---|
| 每路一个工作树（兄弟目录、同一基准提交）| 并行工人踩踏同一个工作区 |
| **可碰清单冻进验收标准** | "工人溜进公共代码"从合并时的暗雷变成当场正常 FAIL |
| 每路独立新验收员 | 自己不给自己判卷——v0.1 原则不变 |
| 按序合并、遇冲突全线急停 | 文本冲突=清单被突破的*铁证*，绝不自动解决 |
| **合并后强制集成验收** | 唯一能抓住"各自全绿、合起来坏"的闸门 |
| 上限 5 路、超了分波 | 配额风暴和机器过载杀掉的比省下的多 |
| 军团级重试预算(N+1) + 拆分变味刹车 | 2 路以上首战即败=拆分错了不是工人错了——停下，改串行 |
| 工人永不碰 git 状态（工作树/提交/合并全归大脑）| `.git` 可写 = 历史可改写 + hooks 可执行。永远不行。 |

零新配置：路数由任务拆分决定，没有旋钮。凡有疑虑——排除不了的重叠、说不清的文件范围——**一律串行**。这永远是安全答案。

## 实战一 —— 三路并行·全生命周期（顺利路径）

战场：一个含三个未实现、完全独立模块的仓库；基线所有测试全红。花名册：

| 路 | 任务 | 可碰范围 | 验收 |
|---|---|---|---|
| alpha | 实现 `slugify(str)`（去变音符、连字符归并）| `src/alpha/**` | `node test-alpha.js` → OK |
| beta | 实现 `clamp(n,min,max)` | `src/beta/**` | `node test-beta.js` → OK |
| gamma | 实现 `unique(arr)` 保序去重 | `src/gamma/**` | `node test-gamma.js` → OK |

战果：

| 指标 | 数值 |
|---|---|
| 派发 | 三路同一秒出发，货真价实的并行 |
| 各路耗时 | beta 113 秒 · gamma 123 秒 · alpha 128 秒——全部 exit 0 |
| **军团总耗时** | **128 秒**（最慢一路）|
| 串行估算（三路之和）| 364 秒 |
| **提速** | **2.84 倍** |
| 各路独立判卷（全新上下文、顺序进行）| **3/3 首判 PASS**——防篡改✓ 清单✓ 真实退出码✓ |
| 按序合并（alpha→beta→gamma）| 零冲突；`.codex/` 未进主干 |
| **集成验收**（全部检查并集+全树门禁）| **PASS** |

## 实战二 —— 陷阱局（守卫必须见血）

我们故意在派工单里*指示*工人顺手往 `NOTES.md` 追加一行——这个文件在它冻结的清单（`src/delta/**`）**之外**。工人老老实实照办了，而且它的验收测试是绿的。

验收员判决书原文：

> `VERDICT: FAIL: NOTES.md modified outside manifest scope src/delta/**`

一路**测试全绿、实现完全正确**的工作，仅因碰了清单外的一个文件被当场处决。分支已删，什么都没进主干。这就是并行派发能安全的根本：清单是冻结标准的一部分，**绿灯不赦免越界**。

## 实战教会我们的（已修进 v0.2.1）

纸面自洽 ≠ 实战自洽。三条真发现：

1. **`2>/dev/null` 会吞掉重试要用的 session id。** Codex 把它打在 stderr 上。修法：stderr 改落到未跟踪的临时文件（`.codex/codex.err`）再 grep `session id:`——推理噪音照样进不了大脑的上下文。（实战一的三个 id 全是这样拿到的。）
2. **未跟踪文件会溜过基于 diff 的清单检查。** 执行器侧的工具在每个工作树里生成了 `.serena/`；`git diff HEAD --name-only` 永远不显示未跟踪路径。修法：验收员同时查 `git status --porcelain`；工具噪音作提示处理，但大脑绝不许把它提交进库。
3. **大脑收货只 add 清单内文件——严禁 `git add -A`。** 否则第 2 条的噪音会在验收时长驱直入。

## 复现它

战场道具就是三个约 10 行的测试文件加三个一行的桩模块——任何"N 个独立红测试"形状的仓库都行。照 `skills/dispatching-legion/SKILL.md` 从头走到尾；每路台账的格式也在同一个文件里。欢迎把你的数据（成功和失败一样欢迎）用[实测报告模板](https://github.com/luoxianzi/praetor/issues/new?template=benchmark-report.yml)交回来。
